# Integrity Engine - Liquid OS (v1)

## Overview
The Liquid OS Integrity Engine is a multi-layered forensic suite designed to detect and deter academic dishonesty. It combines real-time behavioral telemetry with post-submission AI analysis and cross-student similarity detection.

---

### Layer 1: Behavioral Telemetry (Real-time)
The `test-session` application utilizes specialized React hooks to capture telemetry without interrupting the student's workflow:

- **`useBehavioralTracking`**: Monitors keystroke dynamics, Words Per Minute (WPM) consistency, and "Paste" events within the code editor.
- **`useIntegrityListeners`**: Listens for `visibilitychange` and `blur` events to log tab-switching and window focus loss.
- **Heartbeat System**: Ensures telemetry is flushed to the server every 30 seconds to prevent data loss in case of a crash.

#### Integrity Score Deductions
| Event | Weight | Justification |
|-------|--------|---------------|
| **Tab Switch** | -30 points | High probability of AI or search engine usage. |
| **Focus Lost** | -2 points | Distraction or peripheral communication check. |
| **Similarity Flag** | -15 points| Strong evidence of peer-to-peer code sharing. |

---

### Layer 2: AI Forensic Auditor
Logic located in: `server/lib/auditor.js`

Upon submission, the system triggers an asynchronous audit using a local **Ollama** instance (running `deepseek-coder-v2` or similar).

- **Data Inputs**: Total session duration, violation counts, total characters pasted, and the final source code.
- **Analysis**: The AI evaluates if the code quality matches the behavioral patterns (e.g., a perfect snippet pasted in 2 seconds is flagged).
- **Output**:
  - `suspicion_score` (0-100)
  - `narrative`: 2-sentence auditor logic.
  - `primary_flag`: The most significant red flag identified.

---

### Layer 3: Similarity Engine
Logic located in: `server/lib/similarity.js`

To detect collusion, the system uses a **Token-based Jaccard Similarity** model.

1. **Tokenization**: Source code is stripped of identifiers, comments, and strings. Structural tokens (keywords, operators, punctuation) are preserved.
2. **Frequency Mapping**: Both submissions are converted into "bags of tokens".
3. **Similarity Index**: Computes the Jaccard index (`intersection / union`).
4. **Flagging**: Submissions exceeding a configurable threshold (default: **0.85**) are flagged for manual admin review in the `SimilarityFlagsPanel`.

---

### Layer 4: Hardened Frontend Container
- **Clipboard Control**: Code editors use restricted paste handlers to track and limit external content injection.
- **State Partitioning**: Each `test-session` is isolated from other OS windows via the `Zustand` state engine, preventing unintended data leakage between apps.

---

## Admin Forensics UI
Administrators can access the **Integrity App** to view granular forensic reports:
- **Timeline View**: Visual representation of when tab switches and focus losses occurred during the test.
- **Diff View**: Side-by-side comparison for flagged similarity cases, highlighting structural identicals.
- **AI Verdict**: The narrative generated by the AI Forensic Auditor.
